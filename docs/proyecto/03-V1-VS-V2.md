# 03 — v1 (Electron) vs v2 (Go + Wails)

## Resumen

El repositorio contiene **dos generaciones** del producto. El desarrollo **activo** es **v2** en `vantare-v2/`. El monorepo **v1** en `apps/desktop/` + `packages/` es legado funcional pero **no es el objetivo del reinicio**.

---

## Comparativa

| Aspecto | v1 (Electron) | v2 (Wails + Go) |
|---------|---------------|-----------------|
| Runtime | Chromium embebido (~200 MB+) | WebView2 del sistema |
| Telemetría | Node / adaptadores TS | Go mmap + parser offsets |
| Hub UI | React en `apps/desktop/src/renderer/hub/` | React en `vantare-v2/frontend/src/hub/` |
| Overlays | Ventanas Electron + bundles | Una ventana compuesta shrink-wrap |
| OBS | Express + SSE (planificado v1) | Go HTTP+SSE (Fase 6 v2) |
| Estado | Sprints 1–8, temas F1, muchos cambios locales | Fases 0–5 cerradas en master plan |
| Docs | `docs/ARCHITECTURE.md`, `ROADMAP.md` | `docs/V2-*.md`, `docs/proyecto/` |

---

## Qué reutilizar de v1

| Recurso v1 | Uso en v2 |
|------------|-----------|
| `tools/generate-lmu-offsets.py` (o similar en repo) | Codegen → `internal/telemetry/lmu/offsets.go` |
| Fixtures / conocimiento LMU | `vantare-v2/testdata/` |
| Diseño F1 / hub iterations | **No** — canónico es `hub_main_v5.html` |
| `packages/ui-core` componentes | Inspiración; v2 tiene widgets propios en overlay/ |
| Auth Supabase | Portar en fase posterior |

---

## Qué NO mezclar sin decisión explícita

- No mover lógica de telemetría v1 Node a v2 sin pasar por pipeline Go.
- No aplicar estilos F1 del sprint 8 v1 al hub v5 sin aprobación de Isaac.
- No asumir que `pnpm dev` en raíz arranca v2 — v2 usa `go run ./cmd/vantare` desde `vantare-v2/`.

## Estructura v1 (referencia)

```
apps/desktop/          # Electron main + renderer (hub, overlays)
packages/
  ui-core/             # GlassPanel, F1Card, etc.
  sim-core/            # Adaptadores telemetría TS
shared/types/          # IPC types
```

---

## Migración conceptual

v2 **no es un refactor incremental** de Electron: es **reescritura** con objetivos de rendimiento. Las features v1 se **reimplementan** por fases en el master plan v2 (F4 overlay, F5 hub, F6 OBS, F8 temas, F9 multi-sim).

Para nuevas sesiones: **trabajar solo en `vantare-v2/`** salvo documentación o extracción de offsets/fixtures desde v1.
